The story

Origin

Good farmers should not
lose good seasons.

Damilare Odumosu is a third-generation farmer. He grew up watching good farmers lose good seasons — not because they were lazy, not because they were careless, but because they were blind.

A storm came in early and they didn't know to move up harvest. A disease window opened on a Tuesday and they didn't scout until Friday. A yield estimate was off by 30 percent and they were locked into a contract they couldn't fill.

The intelligence existed. The signals were there — in the canopy, in the soil moisture, in the satellite image, in the berry clusters. But nothing connected those signals to a decision in time.

"My life mission: make sure the next generation of farmers does not have to guess."

The Problem

The gap that costs
seasons every year.

Large operations pay for satellite subscriptions, dedicated agronomists, and decision-support software that costs tens of thousands per year. Family farms run on experience, instinct, and luck. CropMind exists to close that gap.

Large operation knows:
  • Disease pressure by block, updated daily
  • Vine water stress from satellite indices
  • Yield estimate ±5% by July
  • Optimal harvest window per variety
  • Historical performance by row
  • Real-time weather risk alerts
Family farm knows:
  • What they see when they walk the rows
  • What they saw last year
  • What the co-op newsletter said
  • What their neighbor told them
  • What the label on the spray can says
  • What it looked like before the storm

The Intelligence Stack

Nine services. One platform.
Built by two people.

Every layer of the CropMind intelligence stack is a production service, deployed on Google Cloud Run, callable from the mobile app or backend pipeline. These are not prototypes. They are running for paying growers right now.

See — What is in the field right now
fruit-detection
Berry Cluster Detection
YOLOv9 model counts berries, identifies clusters, and feeds yield estimation. Fine-tuned on CropMind's annotated vineyard dataset.
Python · FastAPI · Cloud Run
TEED-edge-detection
Vine Structure Edge Detection
Tuned TEED model extracts vine structure edges from field photos — the visual preprocessing layer before object detection.
Python · TEED · Cloud Run
depth-estimation-api
3D Depth for Fruit Counting
Monocular depth estimation (Depth Anything + Depth Pro) separates foreground berries from background foliage to improve counting accuracy.
Python · FastAPI · Cloud Run
Know — The field in context
row-finder-api
Field Row Geometry
FFT-based row detection from aerial imagery — returns row direction, spacing, and count without manual georeferencing.
Python · FastAPI · Cloud Run
aerial-data-visualization-api
Satellite Index Heatmaps
Converts raw GeoTiff satellite imagery into NDVI, NDMI, and NDWI overlays. Makes canopy stress visible across an entire block.
Python · rasterio · Cloud Run
gdd_microservice
Growing Degree Days
Calculates GDD accumulation from temperature data. The underlying engine for stage-crossing alerts and harvest timing predictions.
Python · FastAPI · Cloud Run
Predict — What is coming
brix-forecast
Sugar Prediction & Harvest Timing
Adaptive sigmoid ML model forecasts Brix accumulation over time. Combines historical variety curves with real-time measurements to predict harvest date within days.
Python · FastAPI · scikit-learn · Cloud Run
disease-pest-detection
Disease & Pest Recognition
Vision classifiers identify powdery mildew, botrytis, leafroll, phylloxera, and more from field photos. Returns condition name, confidence, and action tier.
Python · YOLO · Cloud Run (crop33137)
